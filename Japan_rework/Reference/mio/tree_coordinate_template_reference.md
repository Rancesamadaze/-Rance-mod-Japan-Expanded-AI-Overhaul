# HOI4 MIO Tree Coordinate Template Reference

> 目的：整理本项目可直接复用的 15 节点 MIO 树坐标模板，减少后续为模板组织去原版逐个抄骨架的成本。
>
> 范围：本表只处理 trait `position = { x = ? y = ? }` 的骨架分布，不直接规定 parent 连接逻辑、trait 含义、header 分区或数值分配。

## 一、统一约定

- 可用范围：`x = 0 ~ 8`，`y = 0 ~ 4`
- 中轴：`x = 4`
- 坐标均为整数
- 每个模板总节点数：`15`
- 显示比例近似理解为：`x : y = 1 : 1.5`

这些模板的用途是：

- 先确定整棵树的视觉骨架
- 再往上填 parent 关系
- 再分配 branch 语义、trait 名称、图标和 bonus

换句话说，模板只解决“版面长什么样”，不自动决定“树要怎么走”。

## 二、使用原则

### 1. 先选骨架，再写意义

如果当前目标是写项目模板组织，而不是复刻原版 MIO：

- 可以先从本表挑一个坐标模板
- 再按项目需求重建路线含义
- 不需要默认回到原版去抄 node shape

### 2. 坐标模板不等于连接模板

同一套坐标可以配不同 parent 逻辑，例如：

- 单主干推进
- 左右分支并行
- 中途汇合
- 多前置解锁

所以不要因为某个模板长得像“主干树”就强行套固定连接方式。

进一步说，对本项目模板组织默认应这样理解：

- 默认不写 `mutually_exclusive`
- 节点之间的连接关系不由坐标模板预先锁死
- `parent`、`any_parent`、`all_parents` 或更复杂的前置写法，应根据具体树意、路线设计和节点职责来定
- 节点关系（连线）在可行时应尽量保持左右对称
- 节点前置只允许来自上一层，不应跨层取更早节点作为直接前置
- 统一连线规则：中轴上的节点（`x = 4`）应以上一层 `x = 3 ~ 5` 区间内的节点作为 `all_parents`
- 除中轴节点外，其余节点默认使用 `any_parent`

也就是说，坐标模板只提供版面骨架，不自动规定路线封闭关系或分支解锁方式。

### 3. 模板优先服务可读性

选模板时优先考虑：

1. 分层节奏是否清楚
2. 中轴是否稳定
3. 分支是否容易阅读
4. 后续 header 是否容易摆放

不要只因为某个模板“像原版某棵树”就优先选它。

### 4. 模板适合项目内模板组织

这些骨架尤其适合：

- `Rance_model_organization.txt` 这类 archetype/template 组织
- 需要先定 trait 数量与大致分区，再慢慢填 meaning 的工作流
- 不想受原版 specialization 路线语义约束的项目内设计

## 三、当前保留模板总览

当前项目只保留这三套模板作为有效骨架：

| 模板 | 分层节奏 | 主要观感 | 适合的树感 |
| --- | --- | --- | --- |
| B | `1-3-3-5-3` | 中轴最强、主干明显 | 主线推进树 |
| D | `1-2-4-3-5` | 双翼展开、层次鲜明 | 左右分化树 |
| E | `1-4-2-4-4` | 先散后聚再散 | 漏斗整合树 |

废弃说明：

- 模板 A 已废弃
- 模板 C 已废弃
- 模板 F 已废弃

后续新建或重写 MIO 时：

- 只从 `B / D / E` 中选择骨架
- 不再默认从 `A / C / F` 中挑选

## 四、模板 A：自然展开型

分层坐标：

- `y = 0`: `(4,0)`
- `y = 1`: `(3,1)` `(5,1)`
- `y = 2`: `(2,2)` `(4,2)` `(6,2)`
- `y = 3`: `(1,3)` `(3,3)` `(5,3)` `(7,3)`
- `y = 4`: `(0,4)` `(2,4)` `(4,4)` `(6,4)` `(8,4)`

总点集：

```txt
(4,0)
(3,1) (5,1)
(2,2) (4,2) (6,2)
(1,3) (3,3) (5,3) (7,3)
(0,4) (2,4) (4,4) (6,4) (8,4)
```

描述：

- 节奏为 `1-2-3-4-5`
- 从上到下逐层展开，整体最像自然长开的树
- 外轮廓顺滑，后续做左右分化或底层铺开都比较方便

适合：

- 最通用的模板组织骨架
- 需要“越往下选择越多”的树
- 希望玩家一眼就能看懂层级推进关系的树

## 五、模板 B：中央主干型

分层坐标：

- `y = 0`: `(4,0)`
- `y = 1`: `(2,1)` `(4,1)` `(6,1)`
- `y = 2`: `(3,2)` `(4,2)` `(5,2)`
- `y = 3`: `(1,3)` `(3,3)` `(4,3)` `(5,3)` `(7,3)`
- `y = 4`: `(2,4)` `(4,4)` `(6,4)`

总点集：

```txt
(4,0)
(2,1) (4,1) (6,1)
(3,2) (4,2) (5,2)
(1,3) (3,3) (4,3) (5,3) (7,3)
(2,4) (4,4) (6,4)
```

描述：

- 节奏为 `1-3-3-5-3`
- 中轴最强，看起来像一条主干一路向下延伸
- 两侧节点围绕中间展开，天然适合“核心主线 + 两侧辅线”

适合：

- 强主干、弱分叉的树
- 中央节点承担成长、体系、总装或核心路线的树
- 需要让玩家明确感受到“主线推进”的模板

## 六、模板 C：菱形核心型

分层坐标：

- `y = 0`: `(4,0)`
- `y = 1`: `(2,1)` `(4,1)` `(6,1)`
- `y = 2`: `(0,2)` `(2,2)` `(4,2)` `(6,2)` `(8,2)`
- `y = 3`: `(2,3)` `(4,3)` `(6,3)`
- `y = 4`: `(3,4)` `(4,4)` `(5,4)`

总点集：

```txt
(4,0)
(2,1) (4,1) (6,1)
(0,2) (2,2) (4,2) (6,2) (8,2)
(2,3) (4,3) (6,3)
(3,4) (4,4) (5,4)
```

描述：

- 节奏为 `1-3-5-3-3`
- 中间最宽、上下收束，整体像一个菱形核心区域
- 中层承载能力最强，适合把主要 branch 冲突或核心主题放在树中段

适合：

- “中盘最重要”的树
- 需要强调中间一层是大分歧、大展开或大汇合的模板
- 想让视觉重点落在中层而不是底层的树

## 七、模板 D：双翼展开型

分层坐标：

- `y = 0`: `(4,0)`
- `y = 1`: `(3,1)` `(5,1)`
- `y = 2`: `(1,2)` `(3,2)` `(5,2)` `(7,2)`
- `y = 3`: `(2,3)` `(4,3)` `(6,3)`
- `y = 4`: `(0,4)` `(2,4)` `(4,4)` `(6,4)` `(8,4)`

总点集：

```txt
(4,0)
(3,1) (5,1)
(1,2) (3,2) (5,2) (7,2)
(2,3) (4,3) (6,3)
(0,4) (2,4) (4,4) (6,4) (8,4)
```

描述：

- 节奏为 `1-2-4-3-5`
- 左右两翼先展开，中间再收一层，底部再次铺开
- 既有展开感，又不会一路越走越散，层次感比较强

适合：

- 左右翼主题都比较清楚的树
- 中段需要一次回收或汇总的树
- 底层仍想保留较多终端节点的模板

## 八、模板 E：漏斗整合型

分层坐标：

- `y = 0`: `(4,0)`
- `y = 1`: `(1,1)` `(3,1)` `(5,1)` `(7,1)`
- `y = 2`: `(3,2)` `(5,2)`
- `y = 3`: `(1,3)` `(3,3)` `(5,3)` `(7,3)`
- `y = 4`: `(1,4)` `(3,4)` `(5,4)` `(7,4)`

总点集：

```txt
(4,0)
(1,1) (3,1) (5,1) (7,1)
(3,2) (5,2)
(1,3) (3,3) (5,3) (7,3)
(1,4) (3,4) (5,4) (7,4)
```

描述：

- 节奏为 `1-4-2-4-4`
- 上层先分散展开，中间明显收束，然后再向下分化
- 视觉上有“先散后聚再散”的漏斗感

适合：

- 前期多入口，中段做整合，再下放到多个成熟路线的树
- 想在中层放双核心节点或双汇总节点的模板
- 需要明显“合流”感的结构

## 九、模板 F：阶梯收束型

分层坐标：

- `y = 0`: `(4,0)`
- `y = 1`: `(1,1)` `(3,1)` `(5,1)` `(7,1)`
- `y = 2`: `(1,2)` `(3,2)` `(5,2)` `(7,2)`
- `y = 3`: `(2,3)` `(4,3)` `(6,3)`
- `y = 4`: `(3,4)` `(4,4)` `(5,4)`

总点集：

```txt
(4,0)
(1,1) (3,1) (5,1) (7,1)
(1,2) (3,2) (5,2) (7,2)
(2,3) (4,3) (6,3)
(3,4) (4,4) (5,4)
```

描述：

- 节奏为 `1-4-4-3-3`
- 前两层展开较充分，之后逐渐向中轴收束
- 最下层形成较紧凑的收口，终盘感很强

适合：

- 早期给玩家较多方向，后期逐步收拢的树
- 需要把最终路线收束到中轴附近的模板
- “先铺开基础，再压缩成成熟体系”的结构

## 十、怎么选模板

如果还没想清楚，优先按下面思路选：

- 想做最稳妥、最通用的骨架：先选模板 A
- 想做明显主干推进：先选模板 B
- 想把视觉重点放在中层核心区：先选模板 C
- 想做左右双翼分化：先选模板 D
- 想做中途合流再分支：先选模板 E
- 想做前宽后收的阶梯感：先选模板 F

## 十一、落地建议

这些模板进入实际 MIO 编写时，推荐顺序是：

1. 先选模板
2. 再决定每层节点承担什么主题
3. 再补 parent / any_parent / all_parents 逻辑
4. 再决定 header 分区
5. 最后同步 trait 名称、图标和 bonus

和本项目现有流程结合时，建议把这些模板视为：

- `tree_header_position_reference.md` 的配套骨架参考
- `infantry_tank_template_workflow.md` 之后的新起步方式
- `Rance_model_organization.txt` 后续模板作者可以直接复用的版面起点

## 十二、结论

这六套 15 节点坐标模板的价值不在于替代所有原版 MIO，而在于提供一个项目内可复用、可维护、可快速起草的骨架库。

后续如果项目继续扩展：

- 可以继续补 12 节点、16 节点、18 节点模板
- 也可以补“适合多 header 分区”的专用骨架
- 但默认情况下，这六套已经足够覆盖大多数模板组织的首轮草图需求

## 十三、15 节点模板删减指南

本节用于说明：

- 当现成 15 节点骨架太大时
- 如何在尽量保持轴对称和层次感的前提下
- 把模板裁剪到 `9 ~ 14` 节点

统一约定：

- 可用范围：`x = 0 ~ 8`，`y = 0 ~ 4`
- 中轴：`x = 4`
- 坐标均为整数
- 原始模板为 `15` 节点
- 裁剪目标：`9 ~ 14` 节点
- 默认倾向保留更多节点，除非用户明确要求更紧凑的树
- 尽量保持轴对称
- 不采用“从最下层开始整层删除”的方式，以免层次变单薄

### 1. 通用删减思路

15 节点模板裁剪到 `n` 节点时，需要删除：

```txt
d = 15 - n
```

删减时主要使用两类操作。

#### 轴点删除 S

轴点指位于中轴 `x = 4` 上的点。

删除 `1` 个轴点不会破坏左右对称，因此当需要删除奇数个节点时，通常优先删除一个非核心轴点。

轴点删除原则：

- 根节点通常不删
- 不优先删除承上启下的关键主干点
- 优先删除局部过密层中的轴点
- 或删除底部不影响整体轮廓的轴点

#### 对称对删除 P

对称对指关于 `x = 4` 对称的一组两个点。

例如：

- `(0,4)` 与 `(8,4)`
- `(1,3)` 与 `(7,3)`
- `(2,2)` 与 `(6,2)`

删除一组对称对会减少 `2` 个节点，并保持轴对称。

#### 按目标节点数应用

若 `d` 为偶数：

- 删除前 `d / 2` 组对称对 `P`

若 `d` 为奇数：

- 先删除 `1` 个轴点 `S`
- 再删除前 `(d - 1) / 2` 组对称对 `P`

对应到目标节点数：

- `14` 节点：删 `S1`
- `13` 节点：删 `P1`
- `12` 节点：删 `S1 + P1`
- `11` 节点：删 `P1 + P2`
- `10` 节点：删 `S1 + P1 + P2`
- `9` 节点：删 `P1 + P2 + P3`

其中：

- `S1` 表示该模板最推荐优先删除的轴点
- `P1 / P2 / P3` 表示该模板推荐的对称对删除优先级
- 它们不是节点编号

### 2. 总体删减原则

本项目当前统一优先级：

1. 优先删 `y = 3`
2. 再删 `y = 1`
3. 同一层内按“从中轴向两侧”对称删减
4. 只有当 `y = 3` 与 `y = 1` 不足以满足目标节点数时，再考虑其他层

这意味着删减时默认不要先动：

- `y = 0` 的始发节点
- `y = 2` 的中层骨架
- `y = 4` 的终端层

除非目标节点数已经很低，或者某个具体模板存在结构性例外。

优先删除：

1. `y = 3` 层中靠近中轴的对称点，再逐步向外
2. `y = 1` 层中靠近中轴的对称点，再逐步向外
3. 同层中的非关键轴点
4. 只有在前两层不够删时，才处理其他层的枝叶点

尽量保留：

1. 根节点
2. 第二层的主要分支点
3. 能维持中轴感的关键节点
4. 每一层至少保留一定数量节点
5. 模板的标志性轮廓

不推荐：

1. 从 `y = 4` 开始整层删除
2. 先删底部全部节点
3. 先删一侧再删另一侧
4. 为了凑数量破坏左右对称
5. 在还没处理 `y = 3` 与 `y = 1` 之前，先动 `y = 2` 或 `y = 4`
6. 删除承上启下的关键中间节点

### 3. 与连接关系的配合规则

删减规则默认只处理坐标骨架，不自动规定节点间的连接关系。

和本项目模板规则结合时，应同时记住：

- 默认不写 `mutually_exclusive`
- 删除某些节点后，不代表剩余节点必须沿原先想象的路线硬连
- `parent`、`any_parent`、`all_parents` 与更复杂前置，应根据裁剪后的具体结构重新判断
- 连线应尽量保持左右对应和整体对称感
- 直接前置只应取自上一层，裁剪后若原设想会跨层，应重排连接而不是保留跨层前置
- 中轴节点默认改用 `all_parents`，并以上一层 `x = 3 ~ 5` 的节点作为完整前置组
- 非中轴节点默认仍使用 `any_parent`

也就是说：

- 裁的是版面
- 不是固定路线图

### 4. 模板 A：自然展开型的删减

原始结构：

```txt
y=0: (4,0)
y=1: (3,1) (5,1)
y=2: (2,2) (4,2) (6,2)
y=3: (1,3) (3,3) (5,3) (7,3)
y=4: (0,4) (2,4) (4,4) (6,4) (8,4)
```

推荐删减优先级：

- `S1 = (4,3)`
- `P1 = (3,3) + (5,3)`
- `P2 = (3,1) + (5,1)`
- `P3 = (1,3) + (7,3)`

按目标节点数裁剪：

- `14` 节点：删 `S1`，即 `(4,3)`
- `13` 节点：删 `P1`，即 `(3,3) + (5,3)`
- `12` 节点：删 `S1 + P1`，即 `(4,3)`，`(3,3) + (5,3)`
- `11` 节点：删 `P1 + P2`，即 `(3,3) + (5,3)`，`(3,1) + (5,1)`
- `10` 节点：删 `S1 + P1 + P2`，即 `(4,3)`，`(3,3) + (5,3)`，`(3,1) + (5,1)`
- `9` 节点：删 `P1 + P2 + P3`，即 `(3,3) + (5,3)`，`(3,1) + (5,1)`，`(1,3) + (7,3)`

描述：

- 自然展开型在新规则下，先从 `y = 3` 的中轴附近收紧，再回头处理 `y = 1`
- 这样会先压缩中段横向宽度，但保留 `y = 4` 的终端展开感
- 裁剪后仍更容易保持“顶部清晰、底部仍有承接”的视觉节奏

### 5. 模板 B：中央主干型的删减

原始结构：

```txt
y=0: (4,0)
y=1: (2,1) (4,1) (6,1)
y=2: (3,2) (4,2) (5,2)
y=3: (1,3) (3,3) (4,3) (5,3) (7,3)
y=4: (2,4) (4,4) (6,4)
```

推荐删减优先级：

- `S1 = (4,3)`
- `P1 = (3,2) + (5,2)`
- `P2 = (2,1) + (6,1)`
- `P3 = (1,3) + (7,3)`

按目标节点数裁剪：

- `14` 节点：删 `S1`，即 `(4,3)`
- `13` 节点：删 `P1`，即 `(3,2) + (5,2)`
- `12` 节点：删 `S1 + P1`，即 `(4,3)`，`(3,2) + (5,2)`
- `11` 节点：删 `P1 + P2`，即 `(3,2) + (5,2)`，`(2,1) + (6,1)`
- `10` 节点：删 `S1 + P1 + P2`，即 `(4,3)`，`(3,2) + (5,2)`，`(2,1) + (6,1)`
- `9` 节点：删 `P1 + P2 + P3`，即 `(3,2) + (5,2)`，`(2,1) + (6,1)`，`(1,3) + (7,3)`

描述：

- 中央主干型在新规则下仍以保留主干为核心
- 但删减顺序改为先收缩 `y = 3` 与 `y = 1`，而不是先动底层
- 裁剪后依然会保持“中央主干明显，两侧附属分支围绕主干展开”的观感

### 6. 模板 C：菱形核心型的删减

原始结构：

```txt
y=0: (4,0)
y=1: (2,1) (4,1) (6,1)
y=2: (0,2) (2,2) (4,2) (6,2) (8,2)
y=3: (2,3) (4,3) (6,3)
y=4: (3,4) (4,4) (5,4)
```

推荐删减优先级：

- `S1 = (4,3)`
- `P1 = (2,3) + (6,3)`
- `P2 = (2,1) + (6,1)`
- `P3 = (0,2) + (8,2)`

按目标节点数裁剪：

- `14` 节点：删 `S1`，即 `(4,3)`
- `13` 节点：删 `P1`，即 `(2,3) + (6,3)`
- `12` 节点：删 `S1 + P1`，即 `(4,3)`，`(2,3) + (6,3)`
- `11` 节点：删 `P1 + P2`，即 `(2,3) + (6,3)`，`(2,1) + (6,1)`
- `10` 节点：删 `S1 + P1 + P2`，即 `(4,3)`，`(2,3) + (6,3)`，`(2,1) + (6,1)`
- `9` 节点：删 `P1 + P2 + P3`，即 `(2,3) + (6,3)`，`(2,1) + (6,1)`，`(0,2) + (8,2)`

描述：

- 菱形核心型在新规则下会优先压缩 `y = 3` 与 `y = 1`
- 只有在这些层不足以支撑目标节点数时，才开始削弱 `y = 2` 最外缘
- 这样能更久地保留菱形中层的主体感

### 7. 模板 D：双翼展开型的删减

原始结构：

```txt
y=0: (4,0)
y=1: (3,1) (5,1)
y=2: (1,2) (3,2) (5,2) (7,2)
y=3: (2,3) (4,3) (6,3)
y=4: (0,4) (2,4) (4,4) (6,4) (8,4)
```

推荐删减优先级：

- `S1 = (4,3)`
- `P1 = (2,3) + (6,3)`
- `P2 = (3,1) + (5,1)`
- `P3 = (1,2) + (7,2)`

按目标节点数裁剪：

- `14` 节点：删 `S1`，即 `(4,3)`
- `13` 节点：删 `P1`，即 `(2,3) + (6,3)`
- `12` 节点：删 `S1 + P1`，即 `(4,3)`，`(2,3) + (6,3)`
- `11` 节点：删 `P1 + P2`，即 `(2,3) + (6,3)`，`(3,1) + (5,1)`
- `10` 节点：删 `S1 + P1 + P2`，即 `(4,3)`，`(2,3) + (6,3)`，`(3,1) + (5,1)`
- `9` 节点：删 `P1 + P2 + P3`，即 `(2,3) + (6,3)`，`(3,1) + (5,1)`，`(1,2) + (7,2)`

描述：

- 双翼展开型在新规则下，会先把 `y = 3` 的中部收紧，再收 `y = 1`
- 这样不会直接掏空底层，也不会太早破坏最下方双翼展开感
- 只有在需要继续压缩时，才会动到 `y = 2` 的外翼

### 8. 模板 E：漏斗整合型的删减

原始结构：

```txt
y=0: (4,0)
y=1: (1,1) (3,1) (5,1) (7,1)
y=2: (3,2) (5,2)
y=3: (1,3) (3,3) (5,3) (7,3)
y=4: (1,4) (3,4) (5,4) (7,4)
```

特殊说明：

- 模板 E 的原始结构中，除根节点 `(4,0)` 外，没有其他位于中轴 `x = 4` 上的可删轴点
- 因此，模板 E 不适合裁成偶数节点数量

具体来说：

- 适合裁成：`13` 节点、`11` 节点、`9` 节点
- 不建议裁成：`14` 节点、`12` 节点、`10` 节点

原因是：

- 若要在严格保持轴对称的前提下删除奇数个节点，通常需要删除一个中轴节点
- 但模板 E 没有合适的非根中轴点可删
- 强行删除会破坏对称性，或者削弱模板本身的漏斗结构

推荐删减优先级：

- `P1 = (3,3) + (5,3)`
- `P2 = (3,1) + (5,1)`
- `P3 = (1,3) + (7,3)`

按目标节点数裁剪：

- `13` 节点：删 `P1`，即 `(3,3) + (5,3)`
- `11` 节点：删 `P1 + P2`，即 `(3,3) + (5,3)`，`(3,1) + (5,1)`
- `9` 节点：删 `P1 + P2 + P3`，即 `(3,3) + (5,3)`，`(3,1) + (5,1)`，`(1,3) + (7,3)`

描述：

- 漏斗整合型在新规则下，优先压缩 `y = 3` 的靠中节点，再压 `y = 1`
- 这样仍能保留“展开-收束-再展开”的节奏，只是收缩点从中段开始
- 如果目标是 `14`、`12`、`10` 节点，建议不要使用模板 E，改用模板 A、B、C、D 或 F

### 9. 模板 F：阶梯收束型的删减

原始结构：

```txt
y=0: (4,0)
y=1: (1,1) (3,1) (5,1) (7,1)
y=2: (1,2) (3,2) (5,2) (7,2)
y=3: (2,3) (4,3) (6,3)
y=4: (3,4) (4,4) (5,4)
```

推荐删减优先级：

- `S1 = (4,3)`
- `P1 = (1,1) + (7,1)`
- `P2 = (3,1) + (5,1)`
- `P3 = (1,2) + (7,2)`

按目标节点数裁剪：

- `14` 节点：删 `S1`，即 `(4,3)`
- `13` 节点：删 `P1`，即 `(1,1) + (7,1)`
- `12` 节点：删 `S1 + P1`，即 `(4,3)`，`(1,1) + (7,1)`
- `11` 节点：删 `P1 + P2`，即 `(1,1) + (7,1)`，`(3,1) + (5,1)`
- `10` 节点：删 `S1 + P1 + P2`，即 `(4,3)`，`(1,1) + (7,1)`，`(3,1) + (5,1)`
- `9` 节点：删 `P1 + P2 + P3`，即 `(1,1) + (7,1)`，`(3,1) + (5,1)`，`(1,2) + (7,2)`

描述：

- 阶梯收束型在新规则下，先处理 `y = 3`，再处理 `y = 1`
- 这样仍然不会过早削薄底层，但会更快把上中段整理得紧凑
- 当需要继续压缩时，再考虑 `y = 2` 的外侧点
