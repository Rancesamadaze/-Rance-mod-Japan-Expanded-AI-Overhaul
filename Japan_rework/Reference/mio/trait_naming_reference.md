# HOI4 MIO Trait Naming Reference

目的：从原版 `military_industrial_organization_l_simp_chinese.yml` 中提炼出对 trait 命名最有用的规律与词汇，供后续编写 MIO trait 名称时参考。

说明：

- 这不是原版全部 MIO 本地化的搬运。
- 重点是总结“原版怎么命名 trait”。
- 用途是帮助项目内 trait 命名更自然，而不是强行照抄原版。

## 原版命名的常见类型

原版 trait 名称大致分成这几类：

1. 部件 / 构型 / 结构
2. 功能 / 作用 / 任务
3. 工艺 / 生产 / 组织流程
4. 专精 / 专家 / 路线
5. 材料 / 设备 / 附件

后续起名时，优先判断你的 trait 属于哪一类，再决定措辞。

## 一、部件 / 构型 / 结构型

这类名字通常用于表达装备设计本身的结构特征。

原版例子：

- `固定式尾轮`
- `可拆卸机翼`
- `双尾`
- `流线型机身`
- `加固框架`
- `装甲座舱`
- `悬臂式机翼结构`
- `复合货床`
- `钢制货床`
- `开放式底盘`
- `封闭式底盘`
- `首上装甲`
- `加固悬挂`

适合的命名感觉：

- `……构型`
- `……布局`
- `……结构`
- `……设计`
- `……底盘`
- `……机身`

适用场景：

- 机动
- 装甲
- 悬挂
- 车体
- 内部布局

## 二、功能 / 作用 / 任务型

这类名字用于表达装备用途、战术用途、支援能力或任务取向。

原版例子：

- `海事职责`
- `多用途运输机`
- `海军袭击`
- `海军航空巡逻`
- `低空攻击`
- `运输职责`
- `截击机`
- `长程改装`
- `反截击战术`
- `作战组`
- `机枪组`
- `军官支援载具`

适合的命名感觉：

- `……任务`
- `……职责`
- `……支援`
- `……方案`
- `……包`
- `……路线`

适用场景：

- 支援火力
- 反坦
- 突破
- 续航
- 指挥 / 协同

## 三、工艺 / 生产 / 组织流程型

这类名字在原版里非常常见，适合生产节点、标准化节点、工业优化节点。

原版例子：

- `扩建生产设施`
- `长期开发升级`
- `选择性建设`
- `标准化合金`
- `先进生产技术`
- `大规模生产`
- `高效扩展`
- `私营投标`
- `材料回收`
- `废钢熔化`
- `机械组装`
- `快速设计迭代`

适合的命名感觉：

- `……计划`
- `……流程`
- `……标准化`
- `……改进`
- `……生产`
- `……扩展`
- `……优化`
- `……方案`

适用场景：

- build_cost
- 资源节约
- 效率增长
- 效率上限
- 产能

## 四、专精 / 专家 / 路线型

这类名字用于表达某条路线偏向什么方向。

原版例子：

- `战斗机专家`
- `轰炸机专家`
- `舰载飞机专家`
- `战斗机专精`
- `近距支援机专精`
- `战斗机专长`
- `近距支援机专长`
- `轰炸机专精`
- `重型战斗机专精`
- `战术轰炸机专精`

适合的命名感觉：

- `……专精`
- `……专长`
- `……专家`
- `……路线`
- `……取向`

适用场景：

- 主分支节点
- 终端分支节点
- 指向明确用途的节点

## 五、材料 / 设备 / 附件型

这类名字适合小而具体的性能节点，尤其是对某项数值有明确对应关系时。

原版例子：

- `液冷发动机`
- `精制引擎`
- `空气制动器`
- `翼载油箱`
- `炸弹架`
- `高爆破甲炸弹`
- `内部鱼雷挂架`
- `导航设备`
- `高辛烷燃料`
- `加固机翼`
- `鼓型瞄准器`
- `优质材料`
- `反载具地雷`

适合的命名感觉：

- `……设备`
- `……系统`
- `……附件`
- `……油箱`
- `……装甲`
- `……挂架`
- `……燃料`

适用场景：

- 单项属性强化
- 小型中间节点
- 需要和图标高度一致的节点

## 原版常用措辞偏好

从原版中文来看，常见风格有这些特点：

- 喜欢用名词短语，不喜欢整句
- 喜欢“结构 / 方案 / 设计 / 改进 / 专精 / 生产”这种结尾
- 工业节点偏抽象，战斗节点偏具体
- 很少起得特别花，整体偏冷静和工程感

推荐优先使用的结尾：

- `……设计`
- `……布局`
- `……结构`
- `……系统`
- `……方案`
- `……改进`
- `……专精`
- `……生产`
- `……标准化`
- `……计划`

不推荐的倾向：

- 太口语
- 太长的完整句
- 为了文学感而牺牲可读性
- 一个 trait 名同时塞进两个并列主题

## 命名时的实用判断顺序

建议按这个顺序起名：

1. 先判断这个 trait 是设计节点、战斗节点、工业节点，还是组织节点
2. 再判断它更像“结构”还是“用途”
3. 如果是单属性节点，尽量让名字和图标主主题一致
4. 如果是复合节点，名字优先体现最核心的那一个主题
5. 如果是模板节点，优先让名字中性、可复用，不要写得太具体

## 对本项目最有用的落地规则

- 为模板 trait 起名时，优先使用中性工程词，而不是国家风味词。
- 同一棵树中，命名风格尽量统一：
  - 要么偏 `……布局 / ……结构`
  - 要么偏 `……方案 / ……包`
  - 要么偏 `……标准化 / ……生产`
- 如果一个节点主要给 `organization_modifier`，名称可以更像“体系 / 基础 / 计划 / 机制”。
- 如果一个节点主要给 `production_bonus`，名称优先往“标准化 / 生产 / 成本 / 资源 / 效率”靠。
- 如果一个节点主要给 `equipment_bonus`，名称优先往“结构 / 设计 / 配置 / 防护 / 火力 / 机动”靠。

## 适合当前步兵坦克模板参考的词

针对步兵坦克模板，原版风格下比较容易复用的词有：

- `基础`
- `布局`
- `结构`
- `防护`
- `机动`
- `支援`
- `反坦`
- `突破`
- `平台`
- `标准化`
- `生产`
- `资源`
- `储备`
- `方案`
- `计划`

这些词组合起来，通常会比生造长句更像 HOI4 原版 trait 命名。
