## 已查到的关键语法方向

1. 自定义属国等级
   - 原版样例：`common/autonomous_states/sea_warlord_subject.txt`
   - 关键字段：`autonomy_state`、`id`、`is_puppet`、`use_overlord_color`、`rule`、`modifier`、`allowed`、`can_take_level`、`can_lose_level`
   - `has_autonomy_state = autonomy_xxx` 原版有大量用例，特殊属国类型判定应优先用它，再考虑 flag 冗余。
   - `allowed_levels_filter` 可参考 `supervised_state.txt` 和 `nsb_personal_union.txt`。

2. 国策树替换
   - 原版样例：
     - `common/on_actions/03_wtt_on_actions.txt`
     - `common/on_actions/04_mtg_on_actions.txt`
     - `events/BFTB_Greece.txt`
     - `common/decisions/BEL.txt`
   - 关键写法：`load_focus_tree = { tree = xxx keep_completed = yes/no }`
   - 相关触发：`has_focus_tree = xxx`
   - 相关刷新：`mark_focus_tree_layout_dirty = yes`
   - 原版还常在替换后用 `unlock_national_focus = xxx` 修补已有进度。

3. 外观标签、颜色和名称
   - 原版样例：
     - `common/countries/cosmetic.txt`
     - `common/decisions/AUS.txt`
     - `events/AAT_Iceland.txt`
   - 关键写法：`drop_cosmetic_tag = yes` 后接 `set_cosmetic_tag = TAG_xxx`
   - 颜色定义：`color = rgb { ... }`、`color_ui = rgb { ... }`
   - 名称方案：`TAG_autonomy_xxx` 或外观标签本地化二选一/组合。

4. 规划疆域与 collection
   - 原版文档：`common/collections/_documentation.md`
   - 原版样例：`common/collections/collections.txt`
   - 关键写法：`input = constant:state_groups.xxx`、`operators = { limit = { ... } }`
   - 消费者：`collection_size`
   - 阵营/进度方向：`ratio_progress` 见 `common/factions/_documentation.md`
   - 可用于“目标属国规划疆域被日本或盟友控制一半以上”的判定；若比例语法不稳，先写固定半数阈值。

5. 变量、动态修正、联合国策树强化

   - 原版中 `add_dynamic_modifier`、`remove_dynamic_modifier`、变量驱动动态修正用例很多。
   - 下一步要抽取最小样例，特别是动态修正读取变量、刷新和新属国漏挂问题。
