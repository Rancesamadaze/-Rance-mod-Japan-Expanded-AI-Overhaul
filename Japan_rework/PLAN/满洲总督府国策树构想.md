# 满洲总督府国策树构想

记录日期：2026-06-23

本文记录 MAN 作为日本特殊属国体系第一棵专属国策树的构想。当前只作为设计草案和后续实现入口，不代表已经写入玩法脚本。

## 基础定位

- MAN 暂按“满洲总督府”构想，承担特殊属国体系的第一棵样板树。
- 机制层视为“皇国总督府”类型下的 MAN 专属名称与外观；不另立一套和皇国总督府平行的属国类型。
- 这棵树既要体现满洲自己的工业、陆军、边境和资源特色，也要为后续其他特殊属国的专属树定下结构基调。
- 所有国策默认按 35 天设计。实际脚本中通常对应 `cost = 5`；第一阶段磨合节点固定按 `cost = 3` 处理。
- 第一版只设数量上限，不设下限；总国策数最多 22 个，实际数量按最终节点规划裁剪。
- 第二阶段工业发展和第三阶段科研强化可以并列展开。AI 不强制工业优先，工业与科研大致按 6:4 权重推进，保留随机成长感。
- 后创建的属国需要追赶机制。可以考虑让联合国策树的特定进度解锁决议，自动完成、跳过或缩短部分属国国策，但具体语法需要测试。

## 整体阶段

建议首版拆成四个阶段：

1. 磨合与改革。
   - 将 MAN 从旧体系改造成日本帝国体系内的高价值特殊属国。
   - 处理旧国家精神、军队、编制、学说、MIO、设计和共荣圈接入。

2. 工业发展。
   - 用 MAN 特色工业国策提供民用工厂、军用工厂、少量船坞、合成炼油厂和限时开发加成。
   - 标准强度锚点为 35 天提供 2 个工厂；合成炼油厂作为 MAN 资源特色，不受普通工厂锚点限制，可以给得更慷慨。

3. 科研强化。
   - 补科研槽到 5 个。
   - 提供科研加速卡和类别加成 idea。
   - 为每个特殊属国分配擅长科研领域，再通过 AI 策略和科研共享组织放大共荣圈整体收益。

4. 收尾与类型升级。
   - 给 MAN 一个具有国家特色的长期 buff。
   - 根据属国类型提供额外升级，特别是皇国总督府方向的军工贡献机制。

## 第一阶段：磨合与改革

第一阶段主题是“磨合与改革”。目标是把旧满洲国改造成共荣圈内的强力特殊属国，而不是继续沿用原版旧国策、旧军队和旧 AI 逻辑。

### 起点国策

需要一个明确起点，暂定功能：

- 标记 MAN 正式进入满洲总督府专属树。
- 作为后续行政、军队、MIO、共荣圈接入国策的共同前置。
- 可在效果中做少量初始化，例如设置国家 flag、显示说明 tooltip，但不建议把所有初始化都塞进第一个国策。

候选主题名：

- 确立满洲总督府
- 整顿满洲旧政
- 总督府改组令

### 移除旧 IDEA 与授予基础 BUFF

一个国策负责处理旧体系：

- 移除不适合满洲总督府体系继续保留的旧国家精神。
- 授予“皇国总督府”或“满洲总督府”专属基础 buff。
- 基础 buff 可以提供稳定、战争支持、轻度生产、训练或顺从度方向加成。

实现分析：

- 旧 IDEA 清理按“尽量全清”处理，不保留历史转化路线。
- 实装前根据当前生效的 MAN 历史文件、MAN 原国策和本 MOD 可能追加的 MAN 内容，整理一份预计会出现的旧 IDEA 清单。
- 若测试后发现漏网旧 IDEA，再追加到清理脚本中，不为少数漏项提前复杂化机制。
- 基础 buff 最好作为总督府类型通用底座，MAN 自身特色放到第四阶段收尾国策。

### 重组满洲军

一个国策负责军事体系改造：

- 解散旧军队和旧式编制。
- 授予日式基础编制，例如基础步兵师和标准步兵师。
- 刷一定数量的师，提供基础部队。
- 解锁“获得日本编制”的决议。
- 切换学说，并提供足量学说精通度。

实现分析：

- 这一块应做成 scripted effect，避免国策内堆过长脚本。
- 旧部队全删；旧编制能稳删则删，删不干净也不阻塞主流程，因为关键目标是让地图上不再保留旧军队。
- 技术路线不是按模板名称主观猜测，而是先按 `regiments` 的主体或多数 line battalion，也就是编制的 main 组成，筛出“旧编制”部队。原版 MAN 模板的 `infantry`、`cavalry`、`militia`、`irregular_infantry`、`mountaineers`、`light_armor` 等主构成应纳入旧军范围；日式模板以 `Rance_JAP_*` 等特殊兵种作为主体，剩余未命中的模板按日式或新体系模板处理。
- 筛选时以主体或多数 line battalion 为准，不因日式模板里附带少量原版火炮、防空炮、反坦克炮等 line battalion 就误判为旧模板。
- 推荐主路线是“属国专用删除用编制”方案：国策创建一个专门用于清理的临时编制，并解锁或调用 MAN/小属国专用清理决议/效果；有符合条件的旧部队时，先把这些部队统一转成删除用编制，再删除该删除用编制及其部队。
- 这个属国清理效果不能和日本现有 `JAP_ai_cleanup_irregular_divisions_effect` 共用。日本删除“日本基础步兵师”是为了清理自身已经落后的填线师；小属国的基础日式步兵师仍然重要，不能照抄日本脚本去删基础步兵师。

伪代码流程：

```text
重组满洲军国策完成
  创建或确保存在 JAP_create_subject_old_army_cleanup_template
  解锁/显示旧军清理决议
  如果 MAN 仍有旧主构成部队
    将旧主构成部队全部 change_division_template 到 属国旧军清理编制
    delete_unit_template_and_units 或 delete_units 删除 属国旧军清理编制
  创建或确保存在日本基础步兵师、日本步兵师模板
  按中型属国口径刷 15 个日本基础步兵师、5 个日本步兵师
  强制切换到日本 AI 倾向陆军主学说
  子学说与精通度授予先进入测试队列
```

```hoi4
			division_has_majority_template = light_armor
		}
	}

MAN_rework_convert_old_army_to_cleanup_template = {

	every_country_division = {
		limit = {
			OR = {
				division_has_majority_template = infantry

				division_has_majority_template = irregular_infantry

		}
	}

MAN_rework_delete_old_army_cleanup_template = {
	delete_unit_template_and_units = {
		division_template = "属国旧军清理编制"

	}
```
```hoi4
}
```

- 如果 `delete_unit_template_and_units` 对临时编制表现不稳定，退一步使用 `delete_units = { division_template = "属国旧军清理编制" disband = yes }`，然后把旧模板留作无兵空模板；后续再用模板锁定、训练上限或 AI 模板策略避免它被重新训练。
- 若要彻底删除旧模板本身，再额外按 MAN history/OOB 和 MAN 国策可能创建的旧模板名，生成显式 `delete_unit_template_and_units = { division_template = "旧模板名" disband = yes }` 清单；这属于增强清理，不作为旧军消失的唯一依赖。
- 日本自身的旧师清理继续使用日本现有逻辑，不迁移到这个 MAN 清理效果里；MAN 清理效果只负责把 MAN 的旧主构成部队转入“属国旧军清理编制”并删除该临时模板部队。
- `common/scripted_effects/JAP_templates_scripted_effects.txt` 是正式版 `name_overlay` 文件；这个创建效果已经落为 `JAP_create_subject_old_army_cleanup_template`，后续如果把转换/删除也抽成正式脚本效果，应继续放在 overlay 效果里，通用国策只调用效果 ID，不直接散写硬编码模板名。
- 编制名称涉及本地化和语言同步，不能只写硬编码英文名。后续应统一维护模板名、tooltip 和本地化。
- 第 6 项已定：MAN 按中型属国处理，重组满洲军后刷 15 个“日本基础步兵师”和 5 个“日本步兵师”。刷兵参数参考日本现有脚本，使用 `start_experience_factor = 0.3`、`start_equipment_factor = 1.0`；人力按 `create_unit` 创建师时的满员逻辑处理，除非实测发现需要额外补人力。
- 第 8 项替换语法已由作者实测确认：`set_grand_doctrine` 与 `set_sub_doctrine` 可以替换已有学说。正式路线继续按 doctrine 文件内 `ai_will_do` 与 `original_tag = JAP` 判断日本倾向；陆军、海军、特种可叠加作者确认的小国适配，空军不做小国改写，完全沿用日本倾向。如果后续新增显式 doctrine AI strategy，则以该 strategy 为最高优先级。海军、空军、特种扩展组合已进入 debug 决议复核队列。

### 特殊属国 AI 编制接入问题

重组满洲军会触及 `common/ai_templates/`：只给 MAN 创建日式模板还不够，还要确认 MAN 是否能吃到日本 AI 模板文件里的 `target_template` 逻辑。首版路线已定为方案 B：直接把日本 AI 模板文件 `common/ai_templates/templates_JAP.txt` 中的日本模板路由从 `available_for = { JAP }` 改成 `blocked_for` 主要非日本国家，使 JAP 和 MAN 都能看到同一套日式目标模板。

技术边界：

- `ai_templates` 的 `available_for` / `blocked_for` 是 role-level 模板的国家路由层。当前项目写法和原版 `_documentation.md` 都按国家 TAG 列表使用；不要把它当成能直接写 `JAP_rework_is_japanese_special_subject = yes` 的动态触发器。
- 原版文档明确提醒 `available_for` 和 `blocked_for` 二选一；两者都不用或两者都用，行为未定义。因此改路由时必须确保每个模板块都有清晰路由。`JAP_suppression` 也纳入同一套 `blocked_for` 路由，避免无路由块的 undefined 行为。
- 本阶段暂不设计新的 `ai_strategy`。这里只评估“MAN 能不能看到并安全使用日本 AI 模板”的路由问题。
- 日本模板中的主要 role 是 `JAP_basic_infantry`、`JAP_infantry`、`JAP_infantry_tank` 等 `JAP_*` 自定义 role；普通国家默认策略通常不会请求这些 role。这个特点是“放宽模板可见范围”的安全前提之一。
- `SIA` 是原版暹罗 TAG。首版按“只屏蔽英美法德意苏六大国”的思路处理，`CHI`、`PRC`、`SIA` 等非六大国先不屏蔽；若实测污染再追加。

方案 A：独立特殊属国模板包，作为回退方案。

优点是最可控，缺点是维护重复。新建 `templates_JAP_subjects.txt`，只把 MAN 或后续特殊属国 TAG 写进 `available_for`，并给它们一套较小的基础/中级日式模板。

```hoi4
# common/ai_templates/templates_JAP_subjects.txt
JAP_rework_subject_basic_infantry_template = {
	available_for = {
		MAN
	}
	role = JAP_rework_subject_basic_infantry
	upgrade_prio = {
		base = 0
	}
	JAP_rework_subject_basic_infantry_default = {
		upgrade_prio = {
			base = 0
		}
		target_template = {
			regiments = {
				Rance_JAP_A_infantry = 9
			}
			support = {
				engineer = 1
				artillery = 1
			}
			regimental_support = {
				fire_support = 1
				field_guns = 2
			}
		}
	}
}
```

方案 B：把当前日本模板改成屏蔽主要非日本国家，作为首版实装路线。

这是首版采用的省维护路线：把 `templates_JAP.txt` 里当前的 `available_for = { JAP }` 改成 `blocked_for = { ... }`，只屏蔽英美法德意苏六个主要国家，不屏蔽 JAP、MAN、中国系、暹罗或其他小国。这样 MAN 可以看到日本模板；其他普通国家虽然理论上也能看到，但因为它们没有日本特殊兵科技，也通常没有 `JAP_*` role 倾向，理论上不会实际编辑或训练这些模板。

```hoi4
# 当前形态
JAP_basic_infantry = {
	available_for = {
		JAP
	}
	role = JAP_basic_infantry
	upgrade_prio = {
		base = 0
	}
	# ...
}

# 首版实装形态
JAP_basic_infantry = {
	blocked_for = {
		ENG
		USA
		FRA
		GER
		ITA
		SOV
		# 只屏蔽英美法德意苏；其他国家实测出问题再追加。
	}
	role = JAP_basic_infantry
	upgrade_prio = {
		base = 0
	}
	# ...
}
```

方案 B 的当前假设：

- 多数日本模板使用 `Rance_JAP_*` 特殊兵种。没有对应科技/解锁的国家无法正常把现有模板改成这些营种。
- 当前日本战斗模板的 outer/inner `upgrade_prio` 多为 `base = 0`，这支持“AI 不会主动花经验乱改这些模板”的假设。
- 其他国家默认模板策略使用 `infantry`、`motorized`、`armor` 等普通 role，不会自然请求 `JAP_basic_infantry`、`JAP_infantry` 等 `JAP_*` role。
- 因此，若不在本阶段新增特殊属国 `ai_strategy`，MAN 是否会主动训练这些 `JAP_*` role 仍需测试；但至少把模板路由放开后，日本侧创建/授予的日式模板不再和 `available_for = { JAP }` 的目标模板逻辑完全脱节。
- 当前 `blocked_for` 首版屏蔽名单为 `ENG`、`USA`、`FRA`、`GER`、`ITA`、`SOV`；若实测发现其他国家被污染，再追加到屏蔽名单，不改变首版路线。

方案 B 需要重点测试：

- 未被屏蔽的小国是否会在没有 `Rance_JAP_*` 科技时尝试改模板并报错。
- `JAP_*` role 在没有对应 `role_ratio` 时是否完全沉默，还是仍会被 AI template 升级逻辑触发。
- MAN 在获得日本编制后，是否会维持这些模板，而不是回到普通 `infantry` 模板。
- `JAP_suppression` 已补同样的 `blocked_for` 路由；测试时确认其镇压模板没有异常外溢即可。
- `JAP_subject_old_army_cleanup` 已补目标模板，并使用 `Rance_JAP_A_infantry` 而不是普通 `infantry`，避免未屏蔽国家把它当作泛用一营步兵模板；测试时确认它不会被任何国家主动训练，只作为 MAN/小属国旧军清理时的临时转换模板。

国策接入口草案：

```hoi4
focus = {
	id = MAN_rework_reorganize_manchurian_army
	cost = 3

	completion_reward = {
		MAN_rework_convert_old_army_to_cleanup_template = yes
		MAN_rework_delete_old_army_cleanup_template = yes

		JAP_create_basic_infantry_template = yes
		JAP_create_infantry_template = yes
		MAN_rework_start_reorganized_army = yes
		MAN_rework_force_japanese_land_doctrine = yes
	}
}
```

长期通用化方向：

- 首版先按方案 B 走，因为它最省维护，也最符合“日本模板作为共荣圈日式编制底座”的方向。
- 如果方案 B 证明会严重污染非目标国家，再退回方案 A，用独立 `templates_JAP_subjects.txt` 做显式特殊属国模板包。
- 无论走 A 还是 B，本阶段暂不新增特殊属国 `ai_strategy`；等国策树和模板授予流程跑通后，再决定是否需要为特殊属国写 role 倾向。

### 统合 MIO 体系

一个国策负责装备设计和 MIO 接入：

- 为全部 MIO 直接增长 20 级。
- 不限定“当前 MIO”或日本系 MIO；MAN 作为样板中型高价值属国，先使用全局强奖励。
- 解锁“获得日本设计”的决议。

实现分析：

- 第 9 项已定：直接对该国全部 MIO 执行 `every_military_industrial_organization = { add_mio_size = 20 }`，不再走“当前 MIO”或日本系 MIO 目标选择。
- 该强度很高，但 MAN 作为第一棵样板树和高价值属国，可以先作为满洲总督府接入日本军工体系的强奖励。
- “获得日本设计”决议应和特殊属国积分、装备设计、AI 装备设计策略联动。

### 纳入共荣圈体系

第一阶段收尾国策：

- 给国家加上“共荣圈成员”动态变量或标记。
- 表示 MAN 正式被纳入共荣圈体系。
- 可作为联合发展国策树、科研共享、积分结算、动态修正和日本侧收益统计的接入口。

实现分析：

- 如果使用变量，需区分“是否成员”的 0/1 标记和“成员等级/贡献值”等数值变量。
- 如果只是判定身份，country flag 或 scripted trigger 更直观；如果需要动态修正读取，则变量更方便。
- 这个节点适合解锁后续工业、科研和联合发展相关机制。

## 第二阶段：工业发展

第二阶段主题是 MAN 特色工业发展。标准强度按 35 天 2 个工厂估算。

建议方向：

- 南满铁路与大陆补给。
  - 提供基建、铁路、补给中心或地区开发加成。
  - 可附带限时建设速度 buff。

- 奉天兵工体系。
  - 提供军用工厂。
  - 可附带步兵装备、火炮、支援装备生产加成。

- 鞍山钢铁与重工业。
  - 提供军用工厂、钢铁资源、工业科研加速或重工业 idea。

- 新京行政工业。
  - 提供民用工厂和稳定方向加成。

- 大连港与辽东船坞。
  - 提供少量海军船坞、海军基地或修船能力。
  - MAN 不应发展成完整海军属国，但可以支撑日本大陆港口体系。

- 合成燃料计划。
  - 提供大量免费合成炼油厂。
  - 可作为 MAN 的资源和后勤特色之一。

- 总督府开发令。
  - 提供限时建设 buff 或地区开发 buff。
  - 可作为阶段中段或阶段收束节点。

强度分析：

- 普通工厂节点以 2 个工厂为锚点。
- 合成炼油厂节点可以强于普通工厂节点，但应注意炼油厂对橡胶和燃油经济的连锁影响。
- 船坞节点应克制，避免抢走“皇国海军辖地”类型的特色。
- 限时建设 buff 可以用来替代一部分直接给工厂，减少节点数量，同时保留成长感。

## 第三阶段：科研强化

第三阶段可以与工业阶段并列。AI 不强制工业优先，工业与科研大致按 6:4 权重推进，使 MAN 的属国成长顺序保留一定随机性。

### 科研槽国策

设计 3 个科研槽国策，用于保险补到 5 个科研槽：

- 第二研究机构。
  - 若当前科研槽少于 3，则增加 1 个科研槽。
  - 若已经满足条件，则跳过到下一个。

- 第三研究机构。
  - 若当前科研槽少于 4，则增加 1 个科研槽。
  - 若已经满足条件，则跳过到下一个。

- 大陆综合研究院。
  - 若当前科研槽少于 5，则增加 1 个科研槽。
  - 若已经满足条件，则跳过到下一个。

实现分析：

- 后续可用 `bypass` 表达“已满足科研槽则跳过”，使后续节点不被卡住。
- 也可在 `completion_reward` 里用条件判断避免超过 5 个科研槽，但这种方式不会自动减少已花费时间。
- 满洲总督府的最终目标是稳定补到 5 槽，不应无限堆科研槽。

### 科研领域与加速卡

MAN 的科研定位暂定为炮兵与综合工业专精。科研卡应尽量指向具体科技线：反坦克、火炮、防空炮后期科技，以及生产线科技、离散工业科技的后期超前研究卡。防空炮和后期工业科技虽然属于较晚科技，但在 PVP 节奏中经常需要提前完成，可以作为 MAN 科研属国的核心价值。

科研强化国策可以提供：

- 工业科研加速。
- 电子、工程或合成资源科研加速。
- 反坦克、火炮、防空炮、生产线科技或离散工业科技的定向科研加速。
- 步兵装备、支援连或陆军学说科研加速可以作为次要备选。
- 研究类别加成 idea。

属国专长设想：

- MAN 适合定位为炮兵、反坦克、防空炮、综合工业、合成资源或大陆后勤方向的科研属国。
- 未来每个特殊属国都分配一个擅长科研领域，再通过 AI 策略引导其研究。
- 科研共享组织负责把属国专长转化为共荣圈整体收益。

实现分析：

- 科研加速卡适合做一次性奖励。
- 科研类别加成 idea 适合体现长期专长。
- AI 策略需要和科研共享目标配套，否则属国可能拿了加成但研究方向不对。

## 追赶机制构想

本节暂作机制占位。具体方案等联合国策树结构、触发点和共享进度做出来后，再进入语法测试与实装设计。

后创建的特殊属国不应花很久才能追上共荣圈主循环。

可能方案：

- 联合国策树完成特定国策后，解锁属国侧追赶决议。
- 追赶决议自动完成若干早期磨合国策。
- 追赶决议减少部分国策天数。
- 对通用阶段使用 shared focus 或统一脚本效果，减少每个属国重复维护。

待测试点：

- 是否存在稳定效果可以同时缩短所有同 id 国策的时间。
- 多国共用同 id 国策是否会引发引用、完成状态或本地化冲突。
- 后创建属国是“直接补完成早期节点”更稳，还是“给限时 focus time reduction”更稳。

初步判断：

- 对后创建属国，自动完成或 bypass 早期通用节点可能比缩短同 id 国策更稳。
- 若要做统一国策内容，优先考虑 shared focus、统一 scripted effect 和国家 flag，而不是复制大量同 id 普通国策。

## 第四阶段：收尾与总督府升级

第四阶段给 MAN 长期身份和国家特色。

### MAN 特色 BUFF

收尾国策可以参考原满洲国国策树，为 MAN 提供一个具有国家特色的长期 buff。

候选方向：

- 大陆兵工枢纽。
- 满洲重工业核心。
- 关东军协同体制。
- 共荣圈北方屏障。

效果方向：

- 军用工厂产出。
- 工业科研速度。
- 补给消耗或补给范围。
- 陆军组织度、恢复率或训练速度。
- 对苏边境、防御、冬季或恶劣地形适应方向加成。

### 皇国总督府军工贡献机制

根据属国类型获得升级。以皇国总督府为例：

- 皇国总督府基础设想是军工上供 50%，自身保留 50%。
- 额外设计一个动态变量机制，把当前军工数量的一半加入日本的动态变量。
- 日本通过动态修正获得等效军用工厂贡献。
- 这部分不扣除属国自身军工数量，作为强力奖励机制，体现主从军事力量合作。

实现分析：

- 需要月度或周期性重算，而不是一次性加值。
- 为避免重复累计，每个属国应记录自己的上一次贡献值。
- 重算流程应大致为：先从日本总变量扣除该属国旧贡献，再按当前军工数量计算新贡献，最后把新贡献加回日本总变量并记录到属国变量。
- 日本侧动态修正读取总变量，转化为军工产能、军用工厂等效收益或其他军事工业加成。
- 该机制很强，适合放在第四阶段收尾，而不是第一阶段直接开放。

可扩展方向：

- 皇国海军辖地可以贡献船坞或海军工业变量。
- 皇国合作政府可以贡献民工、综合工业或科研共享变量。
- 皇国卫星国可以贡献稳定、资源、驻军或基础经济变量。

## 首版数量建议

为了避免 MAN 树过长，首版只设置上限，不设置下限；总国策数最多 22 个，最终按实际规划节点和分支密度裁剪：

- 第一阶段：5 个左右。
- 第二阶段：6 到 8 个。
- 第三阶段：5 到 6 个。
- 第四阶段：2 到 3 个。

在默认 35 天国策和第一阶段 21 天国策混用的基础上，完整主循环时长按实际节点数计算。若后续加入追赶机制和联合国策树联动，后创建属国可以更快接入中后期体系。

## 已确认事项

- MAN 的正式身份是“皇国总督府”下的 MAN 专属名称与外观，即机制层走皇国总督府，显示层使用满洲总督府。
- 第一阶段国策使用 `cost = 3`；全树只设 22 个国策上限，不设下限。
- 工业与科研并列后，AI 不必强制工业优先；大致按 6:4 权重推进即可。

- MAN 按中型属国处理，重组后刷 15 个“日本基础步兵师”和 5 个“日本步兵师”；刷兵参数参考日本现有脚本，经验 `0.3`、装备 `1.0`。

- 学说处理先定第一步：重组满洲军时强制切换到日本 AI 倾向陆军主学说；当前无显式 doctrine unlock strategy，普通路线按 doctrine AI 权重取 `grand_battleplan`，返峰/标准路线取 `rance_furinkazan`。
- MIO 整合已定：直接让 MAN 全部 MIO 增长 20 级，使用 `every_military_industrial_organization = { add_mio_size = 20 }`。

## 我有把握但建议作者过目的事项

这些内容我可以按现有 MOD 风格直接推进，但最好在正式实装前由作者快速过目一次，避免风格或数值偏离预期。

| 序号 | 内容 | 我的建议设计细节 | 需要过目的原因 |
|---:|---|---|---|
| 1 | 国策树结构 | 第一阶段单线磨合，第二阶段工业与第三阶段科研并列，第四阶段汇合到 MAN 特色 buff 和皇国总督府升级 | 结构本身稳，但会影响国策树视觉布局和完成节奏 |
| 2 | 起点国策 | 起点用于设置 MAN 专属树 flag、解锁后续阶段，并显示“正式进入满洲总督府体系”的说明 tooltip | 起点文案和叙事口吻需要符合你对满洲总督府的定位 |
| 3 | 大脚本拆分方式 | 军队重组、MIO 整合、共荣圈接入、军工贡献重算都拆成 scripted effect，国策内只调用效果和提示 | 技术上更稳，也便于复用；但会增加几个脚本效果 ID |
| 4 | 科研槽处理 | 3 个科研槽国策分别补到 3、4、5 槽，用 `bypass` 跳过已满足的节点，避免超过 5 槽 | 这是最稳的设计，但是否保留“已领先也花时间拿别的奖励”需要你确认 |
