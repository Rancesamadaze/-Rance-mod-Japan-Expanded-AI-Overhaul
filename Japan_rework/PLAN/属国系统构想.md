# 属国系统构想

记录日期：2026-06-18

本文记录一套仍在构思中的全新属国机制。当前只作为设计草案和后续实现入口，不代表已经进入玩法脚本。

## 使用范围

- 未来的史实路线与劳农派路线。
- 目标是让 AI 在征服土地后优先放出傀儡国，而不是把吞并作为主要扩张手段。
- 这些傀儡不沿用普通原版属国定位，而是专门服务于日本扩张体系的特殊属国类型。

## 原版属国痛点

1. 工业上供不足。
   - 原版属国能用 `cic_to_overlord_factor` 与 `mic_to_overlord_factor` 调整民用工厂、军用工厂上供。
   - 暂未看到并列的“海军船坞上供”字段，因此船坞贡献可能需要用动态修正和变量手写模拟。

2. 属国战斗力太弱。
   - 科技、装备质量、编制质量、AI 行为都偏弱。
   - 专门写一个完整的属国通用 AI 成本太高，不适合作为第一实现方案。

3. 普通属国会自己训练和使用垃圾装备。
   - 如果直接放任属国自建军队，容易膨胀出质量低、不可控、吃装备的部队。
   - 这会同时削弱日本供给系统，也会让战线指挥变得混乱。

## 核心设计

建立一套日本专属的特殊属国体系，把属国的“自治建军”改为“日本统筹供给和扩军”。

核心原则：

- 禁止属国自行训练师。
- 通过决议系统，根据属国工业情况给属国发放指定数量、指定编制的师。
- 初始装备率设得很低，避免新建部队卷入属国自己的低质量存量装备。
- 通过另一套决议或循环效果，给属国持续供应所需装备。
- 装备应尽量设定为日本产，以保证质量。
- 因为属国无法自己训练师，持续供给不会造成不可控膨胀。
- 禁止属国向外租借装备，禁止进入国际装备市场，避免玩家通过属国套取装备或刷市场。
- 给这些属国施加强烈生产减益，确保其自身无法有效产出装备。
- 海军可以采用类似思路：属国不正常自建海军，由日本通过决议、事件或脚本效果统一授予、补给或改编。

## 陆军供给循环

初步拆成三层：

1. 属国类型与国家规则。
   - 新建日本专属属国类型。
   - 属国类型负责基础从属关系、战争规则、上供比例、颜色和自治等级等。
   - 国家精神或动态修正负责更细的生产、市场、租借、训练限制。

2. 部队授予决议。
   - 决议检测属国工业、核心地区、战争阶段或日本路线阶段。
   - 触发后在属国生成指定模板部队。
   - `create_unit` 的 `start_equipment_factor` 可以设得很低，后续再由日本供应装备。

3. 装备补给决议。
   - 由日本侧周期性补足属国模板所需装备。
   - 优先考虑 `send_equipment`，因为它能指定 `target`、装备类型与数量，并可用 `old_prioritised = yes` 优先送旧装备。
   - 若需要“日本产、指定质量”的装备，后续要继续验证 `add_equipment_to_stockpile`、装备变体、或先由日本库存发送的可控性。

## 工业与船坞贡献

原版属国类型里已有的直接字段：

- `cic_to_overlord_factor`
- `mic_to_overlord_factor`
- `autonomy_manpower_share`
- `extra_trade_to_overlord_factor`
- `overlord_trade_cost_factor`
- `can_master_build_for_us`

船坞方面，首轮搜索没有发现类似 `nic_to_overlord_factor` 的直接属国字段。后续方案倾向于：

- 在属国侧用严重的 `industrial_capacity_factory`、`industrial_capacity_dockyard`、`production_factory_efficiency_gain_factor` 等减益压制自生产。
- 在日本侧用变量累计“属国船坞贡献”。
- 通过动态修正把该变量映射为日本的 `industrial_capacity_dockyard` 或其他海军工业收益。
- 必要时按属国船坞数量、地区价值或路线阶段分档，而不是逐个船坞精确映射。

本 MOD 已有变量驱动动态修正样例，例如 `common/dynamic_modifiers/zzz_japan_rework_dynamic_modifiers.txt` 中：

- `industrial_capacity_dockyard = JAP_showa_statism_industrial_capacity_dockyard`
- `industrial_capacity_dockyard = JAP_industry_industrial_capacity_dockyard`

这可以作为未来“属国船坞贡献”动态修正的工程参考。

## 限制与防钻空子

需要实现或验证的限制：

- 禁止属国自行训练师。
  - 原版 `events/SEA_Japan.txt` 有 `country_lock_all_division_template = { is_locked = yes desc = ... }` 样例。
  - 原版和本 MOD 决议中也有 `set_division_template_cap` 样例，可作为限制指定模板数量的备选工具。

- 禁止进入国际装备市场。
  - 原版国家精神中有 `rule = { can_access_market = no }` 样例。
  - 后续可用属国专属国家精神或动态机制挂给目标属国。

- 限制租借与装备外流。
  - 原版常见的是 `lend_lease_tension`、`lend_lease_tension_with_overlord` 等门槛修正。
  - 是否存在完全禁止租借的国家规则仍需继续确认。
  - 如果没有直接硬开关，需要用 AI 策略、极端门槛、库存控制、市场禁止和生产压制组合降低漏洞。

- 压制属国自生产。
  - 可考虑使用 `industrial_capacity_factory`、`industrial_capacity_dockyard`、`production_factory_max_efficiency_factor`、`production_factory_efficiency_gain_factor`、`line_change_production_efficiency_factor` 等修正。
  - 数值要足够重，目标是让属国不再成为独立装备生产者。

## 待确认问题

- 属国训练锁是使用全模板锁、指定模板上限，还是两者结合。
- 日本产装备的“质量保证”究竟通过日本库存发送、指定变体入库，还是通过模板和补给链间接保证。
- 船坞贡献是按属国实际船坞数量、总工业分档，还是按地区/路线阶段给固定变量。
- 属国海军是直接授予舰艇、授予生产进度、还是只给日本侧等效海军工业收益。
- 是否需要给不同区域属国分级，例如核心军政属国、普通占领属国、海军基地属国。
- 玩家控制日本时，这套决议是否允许手动扩军，还是仅供 AI 自动使用。
- 如果玩家控制属国，是否需要额外规则防止绕过市场、租借和训练限制。
