# 属国系统构想

记录日期：2026-06-18  
精简日期：2026-06-28

本文件只保留“是否要做”的设计判断。语法、实现路线和长推导放到
`Reference/special_subject_system_syntax_reference.md`。

## 一句话目标

让日本扩张后更倾向建立可用的特殊属国，而不是只吞并土地。

## 已定方向

- 特殊属国使用自定义 `autonomy_state`，不走原版自治升降链。
- 属国类型判定优先使用 `has_autonomy_state = autonomy_xxx`。
- 属国可以自行生产和训练，但用轻量 AI、模板、装备设计和决议把方向拉回日本体系。
- 日本不长期托管属国建军，不做“日本替属国无限刷师”的主循环。
- 旧国家精神、旧国策树和旧外观标签需要在接入时清理，避免污染新系统。
- 不再为每个重要属国做一整棵专属国策树；国家特色优先放在决议、idea、动态修正或脚本效果里。

## 首版范围

- 先以满洲为样板：`皇国镇守府` / `满洲镇守府`。
- 先验证自治等级、显示名、外观标签、图标、决议转换和联合国策树接入。
- 复杂的征服后自动放国、地盘整理、积分商店和全属国 AI，等样板跑通后再拆开决定。

## 联合国策奖励编写风格

后续联合国策默认参考 `联合大本营` 的奖励结构，不把奖励都塞进一个无差别块里。

- 发起者专属奖励：只给点下国策的国家，写在 `completion_reward_joint_originator`。这类奖励可以是三军经验、一次性政治/军事资源、解锁提示、发起者专属 flag，或特定属国类型的额外奖励。
- 共有奖励：发起者和通过 `joint_trigger` 的联合成员都应获得一份。实现时优先抽成同一个 scripted effect，在发起者块和成员块里调用；如果确实所有联合参与者都能无条件吃同一份，也可以使用普通 `completion_reward`。
- 日本排除：相当一部分属国成长奖励默认不让日本吃。写法上用 `NOT = { tag = JAP }` 或特殊属国触发器包住实际效果；日本侧最多拿解锁决议、全局标记、说明 tooltip 或共荣圈统计接口。
- 成员奖励：非发起联合成员写在 `completion_reward_joint_member`，不要和发起者专属奖励混写。成员如果也要吃共有奖励，应调用同一个共有奖励效果，并设置同一套领取 flag。
- 后创建属国补领：凡是会永久改变属国成长状态的共有奖励，都要配一个 0 花费决议入口。决议由 `JAP = { has_completed_focus = ... }` 解锁，用“未领取 flag”判定可用，调用同一个奖励效果并设置同一 flag；AI 权重可以给高值，让后成立属国自动补吃。
- 奖励防重复：每类可补领奖励都要有自己的 `JAP_rework_received_*` flag。国策即时奖励和补领决议必须共用同一 flag，避免成立时吃一次、补领时再吃一次。
- 决议不另做强度：补领决议只追平已完成国策的共有奖励，不给额外强化；如果某奖励只适合发起者，不为后创建属国补领。

## 共荣圈陆军可用的 AI 学说组合

以下组合从已删除的旧 MAN 规划中保留，先作为特殊属国通用候选包。后续可由 `共荣圈陆军` 或其后续决议/脚本效果按属国类型发放。

- 原则：不动态复制日本当前任意学说，而是按日本 AI 倾向和小国适配给固定组合；若后续新增显式 doctrine AI strategy，以 strategy 为最高优先级。
- 陆军主学说：普通路线取 `grand_battleplan`；`Rance_jap_return_to_peak_standard` 路线取 `rance_furinkazan`。
- 陆军子学说：`commandos`、`field_engineering`、`streamlined_deployment`、`defensive_postures`。这是旧 MAN 规划里确认的小国方案，其中 `field_engineering` 取代 `air_cavalry`，`defensive_postures` 取代 `infiltration_tactics`。
- 海军组合：`new_convoy_raiding`，以及 `patrol_boats`、`light_task_forces`、`battleship_antiair_screen`。
- 空军组合：`new_operational_integrity`，以及 `air_subdoctrine_fighter_central_field`、`air_subdoctrine_naval_torpedo_tactics`、`air_subdoctrine_long_range_escort`、`air_subdoctrine_flying_fortresses`；空军不做小国改写，沿用日本倾向。
- 特种组合：`special_forces_quality`，以及 `jap_rance_air_sea_joint_assault`、`jap_rance_mountain_ranger_assault`；实装时两个日式特种分支分别写入 `track = 0` / `track = 1`。
- 精通度：旧 debug 草案按目标子学说 `add_mastery amount = 1000` 顶满测试，正式落地时可再调；目前设计倾向保留为强奖励。
- 范围：先给 MAN / `皇国镇守府` 样板验证；是否让所有特殊属国共享，或按 `皇国海军辖地`、`皇国合作政府`、`皇国卫星国` 拆不同包，等首版跑通后再定。

## 暂定属国类型

| 类型 | 用途 |
|---|---|
| 皇国海军辖地 | 港口、群岛、船坞、海军基地价值高的地区 |
| 皇国合作政府 | 大型地区政权，均衡发展 |
| 皇国镇守府 | 靠近日本或陆战价值高的军事属国 |
| 皇国卫星国 | 外围缓冲、小国、低优先级政权 |

## 明确不做

- 不禁止属国自行训练部队。
- 不把属国生产能力压到接近零。
- 不让日本定期替所有属国补齐全部装备缺口。
- 不为每个属国维护大型专属国策树。
- 不在首版解决所有原版专属 AI 残留。

## 待审核

- 四种属国类型是否都保留，还是先只保留 `皇国镇守府`。
- 是否真的需要“积分系统”，还是先用普通决议和脚本效果解决装备/编制同步。
- 属国海军补给先暂缓设计：属国海军产能普遍有限，舰队结构不宜直接套用日本的半年度大舰队发放；后续应按属国类型拆分，只有 `皇国海军辖地` 或其他海军特化属国考虑大型舰队，普通属国优先考虑护航、破交、防卫或岸基支援，等海军属国内容定型后再补海军积分和兑换决议。
- 联合国策树是否保留为主成长系统。
- `共荣圈陆军` 是否只开启陆军主/子学说，还是作为完整 AI 学说包入口。
- 目标子学说精通度是否沿用 `amount = 1000` 的强奖励口径。
- 属国颜色是否每种类型固定一套色系，还是按 tag 单独定。
- 战争中新占领地区是否允许自动建立属国，还是只做玩家手动决议。
- 属国间地盘整理是否进入本系统，还是先完全不做。
- 玩家控制属国时是否允许使用这些辅助决议。

## 国策图标风格参考

- 后续为特殊属国系统、联合国策树或相关日本侧国策生成图标时，优先参考 `PLAN/japan_vanilla_focus_icon_style_collage.png`。这张拼图收集的是原版日本国策中较适合作为风格样本的图标，主要参考其金属边框、日章/菊纹、建筑、人物肖像、旗帜与暖色高光的构图和笔触。
