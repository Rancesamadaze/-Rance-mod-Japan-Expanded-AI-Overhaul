# 脚本登陆计划

## 目的

本文记录皇道派军事 AI 中已经存在或已经规划落地的登陆链，用于后续把 AI 的链式登陆思路改写为“大本营——登陆作战指令”决议组内的脚本登陆决议。

核心思路：

- AI 登陆链中的“出发州”只是启停门槛，不代表 HOI4 AI 能被指定真实出发港。
- 脚本登陆可以把“控制起点 A 后，对终点 B 执行登陆 + 敌军伤害”明确化。
- 现有可用组合效果为三档登陆乘两档伤害：
  - `JAP_void_landing_small_with_small_damage`
  - `JAP_void_landing_small_with_big_damage`
  - `JAP_void_landing_medium_with_small_damage`
  - `JAP_void_landing_medium_with_big_damage`
  - `JAP_void_landing_big_with_small_damage`
  - `JAP_void_landing_big_with_big_damage`

## 总体规则

皇道派登陆链通常由四个部件组成：

1. `reversed = yes` 的 `invade`，由目标州控制者触发，要求目标控制者与日本交战。
2. 日本侧的 `invasion_unit_request`，给目标州增加登陆兵力需求。
3. 共享登陆窗口，例如 `naval_invasion_focus` 与 `naval_invasion_dominance_weight`。
4. 登陆后 30 天桥头堡推进，用国家 flag 与州 flag 触发 `front_unit_request` 和高优先级 `front_control`。

转为脚本登陆时，优先保留第 1、2 条的链式门槛，把第 3、4 条作为 AI 后续接管逻辑，而不是在脚本效果里复制 AI 策略。

登陆规模规划原则：

- 小型登陆：用于小岛、外岛、群岛跳板、末端清理目标，重点表现抢占港口和据点，不模拟大兵团正面登陆。
- 中型登陆：用于大岛、半岛、海峡门户、区域主桥头堡和普通大陆沿岸开口，是东南亚、南亚、东印度-澳新、太平洋主节点、夏威夷和远东大陆登陆的默认规模。
- 大型登陆：目前只用于对美国本土级别的登陆行动。除北美西海岸合并登陆决议外，不把勃固、夏威夷、澳大利亚或远东大陆方向升级为大型登陆。
- 伤害档位与登陆规模分开决定：规模表示投入兵力和登陆容量占用，伤害表示对守军组织与兵力的冲击。

登陆容量规划原则：

- 全局登陆容量只使用 `JAP_scripted_landing_capacity_available = yes` 检查 `JAP_naval_invasion_capacity > 0`。
- 不按小/中/大型登陆分别检查 `>= 1`、`>= 3`、`>= 5`；允许一次中型或大型登陆把 `JAP_naval_invasion_capacity` 扣成负数。
- 一旦容量被扣成 0 或负数，新脚本登陆决议不再可用，直到正在读条或冷却的行动返还容量。

AI 优先级规划原则：

- 批量脚本登陆决议的 `ai_will_do` 均不低于 `500`。
- 粗略按地理距离和作战纵深排序：离日本本土越近、越靠前的作战门槛，AI 权重越高；阶段门槛负责限制时序，AI 权重只用于同一阶段内的取舍。
- 建议权重：冲绳/台湾/菲律宾、东南亚门户为 `850-950`；南亚、东印度门户、太平洋内外岛链为 `700-850`；东印度第二波、澳新、远东大陆、夏威夷为 `600-750`；北美西海岸合并决议因距离最远，默认 `500-650`，但必须通过“筹备美国本土登陆”门槛后才可用。

## 登陆链总览

### 1. 对华沿海登陆

本轮不纳入“大本营——登陆作战指令”，不生成山东、青岛、上海、广州方向的常规登陆或奇袭登陆。

备注：

- 既有皇道派 AI 的真实登陆后桥头堡推进逻辑可以保留；它只负责 AI 真实登陆后的短期接管，不属于这套脚本登陆决议。
- 若未来重新评估对华沿海方向，应单独开方案，不从本轮批量决议中继承。

### 2. 东南亚登陆链

阶段：中国战事基本结束后，东南亚核心走廊尚未完成。

链条：

- 海南 `591` -> 暹罗 `289`
- 海南 `591` -> 马来亚 `336`
- 马来亚 `336` -> 勃固 `995`

备注：

- 新加坡 `1021` 是核心走廊完成条件，不是当前主动登陆目标。
- 勃固 `995` 同时是后续南亚登陆链的重要出发门槛。

AI 数值：

- 暹罗 `289`、马来亚 `336`、勃固 `995`：`invasion_unit_request = 60`
- 登陆窗口：`naval_invasion_focus = 100`，`naval_invasion_dominance_weight = 40`
- 制海限制：`enemies_naval_strength_ratio < 1.5`
- 登陆后 30 天桥头堡：`front_unit_request = 30`，`front_control priority = 700`

脚本登陆建议：

- 暹罗、马来亚：中型登陆 + 小伤害。
- 勃固：中型登陆 + 大伤害；它是打开印度方向的关键门槛，但仍按中型登陆处理。

- 法属印度和果阿不作为第一批主动登陆目标。

AI 数值：

- 登陆窗口：`naval_invasion_focus = 100`，`naval_invasion_dominance_weight = 40`

### 4. 东印度-澳新枢纽登陆网络

阶段：东南亚核心走廊完成后，由马来亚/新加坡接替东南亚军区。

性质：枢纽网络，不是单线链。马六甲-新加坡开门后，逐步打开东印度、澳大利亚、新西兰。

并发限制规划：

- 东印度-澳新军区需要一套独立于全局奇袭计数的军区并发计数系统，确保该军区内同时处于 30 日读条/冷却中的脚本登陆行动不超过 3 个。
- 建议变量：`JAP_scripted_landing_eia_active_plans` 记录当前东印度-澳新军区正在读条或冷却的脚本登陆行动数量，`JAP_scripted_landing_eia_active_plan_limit = 3` 记录上限。
- 建议触发器：`JAP_scripted_landing_eia_plan_slot_available = yes`，用于检查 `JAP_scripted_landing_eia_active_plans < JAP_scripted_landing_eia_active_plan_limit`。
- 建议效果：常规登陆计划被选用且进入 30 日筹备时调用 `JAP_scripted_landing_eia_reserve_plan_slot = yes`，奇袭被选用且进入 30 日冷却时同样调用该效果；读条完成或取消时调用 `JAP_scripted_landing_eia_release_plan_slot = yes`。
- 奇袭虽然在选择决议时立即发动，但仍占用东印度-澳新军区并发槽，并按行动规模占用全局登陆整备能力，直到 30 日读条/冷却结束后释放；同时仍应受全局脚本登陆开关、海军力量门槛和登陆计划容量门槛限制。

第一波门户：

- 马来亚 `336` 或新加坡 `1021` -> 苏门答腊 `672`
- 马来亚 `336` 或新加坡 `1021` -> 爪哇 `335`
- 马来亚 `336` 或新加坡 `1021` -> 沙捞越 `333`
- 马来亚 `336` 或新加坡 `1021` -> 加里曼丹 `334`

第二波东印度枢纽：

- 任一合理门户 -> 小巽他群岛 `667`
- 任一合理门户 -> 东帝汶 `721`
- 任一合理门户 -> 摩鹿加群岛 `668`
- 任一合理门户 -> 苏拉威西 `673`
- 任一合理门户 -> 西巴布亚 `669`
- 任一合理门户 -> 巴布亚 `523`

第三波澳新登陆：

- 东印度/新几内亚桥头堡 -> 北领地 `520`
- 东印度/新几内亚桥头堡 -> 西澳大利亚 `522`
- 东印度/新几内亚桥头堡 -> 北昆士兰 `872`
- 东印度/新几内亚桥头堡 -> 新南威尔士 `285`
- 澳洲东岸桥头堡 -> 北岛 `284`
- 澳洲东岸桥头堡 -> 南岛 `723`
- 新南威尔士 `285` 或维多利亚 `517` -> 塔斯马尼亚 `518`

AI 数值：

- 主门户/主桥头堡：`invasion_unit_request = 45`
- 次级枢纽：`invasion_unit_request = 35`
- 小岛或末端岛：`invasion_unit_request = 25`
- 登陆窗口：`naval_invasion_focus = 80`，`naval_invasion_dominance_weight = 40`
- 制海限制：`enemies_naval_strength_ratio < 1.5`
- 主要桥头堡登陆后：`front_control priority = 700`
- 小岛/末端岛登陆后：低强度 followup

脚本登陆建议：

- 第一波门户：中型登陆 + 小伤害。
- 小巽他群岛、东帝汶、摩鹿加群岛、南岛、塔斯马尼亚：小型登陆。
- 苏拉威西、西巴布亚、巴布亚、北领地、西澳大利亚、北昆士兰、新南威尔士、北岛：中型登陆。
- 东印度-澳新方向不使用大型登陆；澳大利亚目标按地域辽阔但防线松散的区域桥头堡处理。

备注：

- 夏威夷方向晚启用，不应在菲律宾和日方岛链仍危险时高权重直冲。

- 冲绳、台湾、马尼拉、吕宋、北棉兰老、达沃：中型登陆。
- 夏威夷：中型登陆 + 大伤害；即使作为北美前置终点，也不升级为大型登陆。

### 6. 北美西海岸合并登陆决议

阶段：四个亚洲主战区基本完成、对美国战争仍在、完成大本营决议“筹备美国本土登陆”、夏威夷 `629` 完全控制且日本本土安全。

决议门槛：

- 必须已完成大本营决议“筹备美国本土登陆” `JAP_imperial_headquarters_prepare_us_mainland_invasion`，即拥有完成 flag `JAP_us_mainland_invasion_preparation_completed_flag`。
- 仍应要求对美国交战、夏威夷 `629` 由日本或附庸完整控制，且日本本土安全。

合并方向：

- 夏威夷 `629` -> 加利福尼亚 `378`
- 夏威夷 `629` -> 俄勒冈 `385`
- 西海岸囤兵转移优先级：加利福尼亚 `378` > 俄勒冈 `385` > 华盛顿 `386`

脚本决议规划：

- 北美西海岸两个登陆方向后续集成到同一个“大本营——登陆作战指令”决议中，而不是拆成两个独立决议。
- 建议决议名口径为“夏威夷方向登陆美国西海岸”，效果中分别用条件块处理加利福尼亚与俄勒冈：目标仍由敌方控制时才执行对应登陆效果，已经被日本或盟友控制的目标跳过。
- 若两个目标都已由日本或盟友控制，决议不应可见或可用。
- 常规登陆计划仍只占用一次决议冷却；当前规模规划为加利福尼亚主攻使用大型登陆，俄勒冈侧翼使用中型登陆。

AI 数值：

- 加利福尼亚 `378`：`invasion_unit_request = 80`
- 俄勒冈 `385`：`invasion_unit_request = 60`
- 登陆窗口：`naval_invasion_focus = 100`，`naval_invasion_dominance_weight = 45`
- 登陆后 30 天桥头堡：`front_unit_request = 35`，`front_control priority = 720`

脚本登陆建议：

- 加利福尼亚：大型登陆 + 大伤害。
- 俄勒冈：中型登陆 + 大伤害，作为美国本土侧翼登陆。
- 脚本决议必须与“筹备美国本土登陆”flag 绑定，避免跳过太平洋链和大本营进军美洲准备。

## 登陆作战与奇袭作战计数设计

脚本登陆决议分为两类：

- 登陆计划：选择时发出登陆预警并占用登陆计划容量，读条结束后使用小伤害组合效果；只有成功登陆才累计一次常规登陆作战计数。
- 奇袭作战：选择时立刻使用大伤害组合效果，并重置登陆作战计数。

变量设计：

- `JAP_scripted_landing_since_last_raid`：记录上一次奇袭作战后，已经执行过多少次登陆作战。
- `JAP_scripted_landing_raid_threshold`：记录当前难度下触发下一次奇袭作战所需的登陆作战次数。
- `JAP_scripted_landing_raid_remaining`：只用于决议 tooltip 显示，记录距离下一次奇袭还差几次成功的常规登陆作战。
- `rance_japan_scripted_landing`：开局自定义游戏规则，控制玩家和 AI 是否允许使用脚本登陆；默认开启，关闭时脚本登陆决议不可见。
- `JAP_naval_invasion_capacity`：帝国海军系列动态修正使用的登陆计划容量；脚本登陆会在常规登陆筹备期间暂时占用该变量。
- `JAP_scripted_landing_eia_active_plans` / `JAP_scripted_landing_eia_active_plan_limit`：东印度-澳新军区专用并发计数，未来用于限制该军区内同时读条或冷却的脚本登陆行动不超过 3 个；常规登陆和奇袭都占用该槽位，读条结束后释放。

判定方式：

- 决议被选用时立刻判定 `JAP_scripted_landing_since_last_raid` 是否达到 `JAP_scripted_landing_raid_threshold`。
- 若未达到阈值，本次为登陆计划：选择决议时弹出登陆预警事件并设置本决议的待执行 flag；读条 30 天结束后，如果起点和目标仍满足条件，再执行登陆效果、让 `JAP_scripted_landing_since_last_raid` 增加 1，并刷新 `JAP_scripted_landing_raid_remaining`。
- 若已经达到阈值，本次为奇袭作战：选择决议时立刻执行登陆与大伤害效果，并弹出奇袭事件，然后将 `JAP_scripted_landing_since_last_raid` 重置为 0；决议仍读条 30 天作为冷却，读条结束不再执行额外效果。

已落地复用触发器/效果：

- 奇袭作战判定统一使用 `JAP_scripted_landing_raid_ready = yes`。
- 脚本登陆开关统一使用 `JAP_scripted_landing_enabled = yes`，样板中放在决议 `visible` 内。
- 登陆计划容量可用判定统一使用 `JAP_scripted_landing_capacity_available = yes`，样板中放在决议 `available` 内；常规登陆与奇袭都需要该条件。
- 制海可用判定统一使用 `JAP_scripted_landing_naval_strength_available = yes`，当前按难度动态调整：EASY 要求 `enemies_naval_strength_ratio < 1`，NORMAL/HARD 要求 `< 1.5`，LUNATIC/EXTRA 要求 `< 2`，PHANTASM 不受此条件限制。
- 登陆计划选择时只设置本决议专用待执行 flag，不增加计数。
- 登陆计划选择时按规模暂时占用登陆计划容量：小规模使用 `JAP_scripted_landing_reserve_small_capacity = yes` 扣除 1，中规模使用 `JAP_scripted_landing_reserve_medium_capacity = yes` 扣除 3，大规模使用 `JAP_scripted_landing_reserve_large_capacity = yes` 扣除 5。
- 登陆计划读条完成或被取消时，按规模使用 `JAP_scripted_landing_release_small_capacity = yes`、`JAP_scripted_landing_release_medium_capacity = yes`、`JAP_scripted_landing_release_large_capacity = yes` 返还容量。
- 登陆计划读条完成时先检查本决议专用待执行 flag，再执行登陆效果并清除该 flag。
- 登陆计划读条完成且起点/目标条件仍满足时，使用 `JAP_scripted_landing_record_successful_regular_landing = yes` 记录一次常规登陆并刷新 `JAP_scripted_landing_raid_remaining`。
- 普通海军登陆成功时，`common/on_actions/JAP_rework_ai_landing_on_actions.txt` 的 `on_naval_invasion` 会检查登陆者是否为日本；若是，则同样使用 `JAP_scripted_landing_record_successful_regular_landing = yes` 记录一次计数。
- 奇袭作战选择时使用 `JAP_scripted_landing_reset_raid_counter = yes`，不设置待执行 flag，使 30 天读条结束时不再执行额外效果。
- 难度事件或其它逻辑修改 `JAP_scripted_landing_raid_threshold` 后，使用 `JAP_scripted_landing_refresh_raid_remaining = yes` 刷新 tooltip 显示。
- 登陆计划读条期间，如果出发地不再由日本、附庸或阵营成员完整控制，则取消决议并清除本决议专用待执行 flag，不增加计数。

难度阈值：

- EASY：15
- NORMAL：10
- HARD：7
- LUNATIC：5
- EXTRA：4
- PHANTASM：3

## 动态地块名事件经验与登陆通知事件

此前动态州名实验已经确认可行：

- 在目标州作用域执行 `save_event_target_as = JAP_debug_dynamic_state_text_target`。
- 随后触发同一个 `country_event`。
- 事件本地化中使用 `[JAP_debug_dynamic_state_text_target.GetName]`，可以按不同决议传入的州动态显示不同州名。

本次脚本登陆沿用该经验，但改用正式事件目标名：

- `JAP_scripted_landing_origin_state`：保存本次脚本登陆的起点州，用来表现日军由何处方向集结或发起行动。
- `JAP_scripted_landing_target_state`：保存本次脚本登陆的目标州。

正式事件：

- `JAP_imperial_headquarters_events.4`：侦测到日军登陆企图。
- `JAP_imperial_headquarters_events.5`：遭受日军奇袭。

触发约定：

- 如果判定为登陆计划，未来决议在被选用时保存起点州为 `JAP_scripted_landing_origin_state`、目标州为 `JAP_scripted_landing_target_state`，并给目标州控制者触发 `JAP_imperial_headquarters_events.4`，提示日军从何处方向集结、即将登陆何处。
- 如果判定为奇袭作战，未来决议在被选用时立刻保存起点州为 `JAP_scripted_landing_origin_state`、目标州为 `JAP_scripted_landing_target_state`，执行登陆与 damage 效果，并给目标州控制者触发 `JAP_imperial_headquarters_events.5`。
- 两个事件都只有确认选项，不附带额外效果。
- 事件文本中起点州名使用黄色：`§Y[JAP_scripted_landing_origin_state.GetName]§!`；目标州名使用红色：`§R[JAP_scripted_landing_target_state.GetName]§!`。
- 后续编写具体决议时，需要确认决议是否存在可靠的“选用时”效果入口；若没有，则把预警事件接入最接近决议开始执行的可用效果块。

参考调用骨架：

```txt
起点州 = {
	save_event_target_as = JAP_scripted_landing_origin_state
}
目标州 = {
	save_event_target_as = JAP_scripted_landing_target_state
	controller = {
		country_event = { id = JAP_imperial_headquarters_events.4 }
	}
}
```
