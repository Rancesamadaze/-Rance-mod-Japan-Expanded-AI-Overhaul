# Doctrines Directory Notes

## Scope

Applies to files under `common/doctrines/`.

## Required References

- Doctrine concepts and generated modifiers: `common/doctrines/_documentation.md`
- Grand doctrine syntax: `common/doctrines/grand_doctrines/_documentation.md`
- Track syntax: `common/doctrines/tracks/_documentation.md`
- Subdoctrine syntax: `common/doctrines/subdoctrines/_documentation.md`
- Project doctrine authoring notes: `common/doctrines/doctrine_authoring_reference.md`
- Modifier lookup: `Reference/modifier_trans.md`
- Localization formatting: `Reference/localisation_formatting_reference.md`

## Operating Rules

- For Rance-exclusive grand doctrines, use `common/doctrines/grand_doctrines/rance_grand_doctrines.txt` as the shared file for land, naval, and air entries unless the user explicitly asks for a different layout.
- Keep AGENTS files short and routing-oriented; put doctrine skeletons, examples, and longer workflow notes in reference files.
- Preserve UTF-8 without BOM and LF for doctrine scripts and Markdown references unless an existing file proves otherwise.
